# 检查清单

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有 Codex App 图标角色。
- 角色保留白色圆角方块、蓝紫云朵和 `>_` 命令符号。
- 角色承担核心动作，不只是装饰。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝紫色主要用于 Codex App 图标角色和少量系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 角色变成黑色实心小怪物、机器人、表情包或可爱卡通。
- 看不出白色圆角方块、蓝紫云朵和 `>_` 命令符号。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 App 截图或完整 UI 界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有隐喻。
- 出现未要求的第三方平台标志。

## 迭代方法

- 太普通：让角色成为动作主体，加入一个奇怪但成立的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 serious, useful, not cute, not mascot。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 角色不像 Codex App：强调 white rounded-square app tile, blue-purple cloud core, terminal glyph ">_"。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得“有点怪”，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是白纸上的手绘产品草图，就不合格。
